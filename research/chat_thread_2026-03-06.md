## 3. AI Provider Discussion

### Options Discussed

**Claude API (Anthropic):**
- Excellent at document understanding, structured data extraction, reasoning
- Token-based pricing, ~$0.01-0.05 per invoice with Sonnet
- Newer ecosystem, fewer pre-built integrations

**OpenAI GPT-4o:**
- Strong multimodal capabilities, largest ecosystem
- Similar pricing, more community resources

**Multi-Provider:**
- Not locked into one vendor, best model per task
- More engineering work upfront

### Recommendation
Start with one provider but design the code so swapping is easy. Put all AI calls behind a service interface, keep prompts separate from API logic. The provider matters less than prompt engineering and validation logic - that's the real IP.

## 4. Claude API Deep Dive (Generated PDF: Claude_API_AP_Integration_Guide_2026.pdf)

User asked for deep research on how Claude API would work, including professional prompt engineering examples. Generated a comprehensive 14-section PDF.

### How Claude Processes Invoices
1. **Document Ingestion**: Convert invoice to base64 or upload via Files API
2. **Visual Understanding**: Semantic layout analysis (not just OCR character reading)
3. **Structured Extraction**: Maps visual data to predefined JSON schema via Structured Outputs
4. **Confidence & Validation**: Flags uncertain fields, detects anomalies, routes to human review

### Model Selection Strategy
- **Haiku 4.5** ($1/$5 per MTok): Simple standard invoices, ~80% of volume, ~$0.0035/invoice
- **Sonnet 4.6** ($3/$15 per MTok): Complex invoices, exception validation, ~$0.01/invoice
- **Opus 4.6** ($5/$25 per MTok): Multi-step reasoning, approval decisions, ~$0.018/invoice

### Cost Analysis
- Typical invoice: ~2,334 tokens total (image + prompt + output)
- With tiered model strategy (80% Haiku, 15% Sonnet, 5% Opus) + caching + batch:
  - 10,000 invoices/month: ~$21/month
  - 100,000 invoices/month: ~$213/month
- Batch API: 50% discount for async processing
- Prompt caching: 90% savings on repeated system prompts

### 6 Professional-Grade Prompts Created
1. **Core Invoice Extraction** - Detailed rules for dates, amounts, vendors, line items, confidence scoring, flags
2. **Autonomous GL Code Prediction** - Chart of accounts integration, vendor history, coding rationale
3. **Fraud & Anomaly Detection** - Duplicate detection, amount/vendor/timing anomalies, construction-specific red flags
4. **AIA G702/G703 Progress Billing** - Full extraction of Application for Payment and Continuation Sheet fields
5. **Lien Waiver Processing** - Classification (4 waiver types), validation, compliance checking
6. **Three-Way PO Matching** - Invoice vs PO vs receiving report with tolerance rules and match decisions

### Key API Features for AP
- **Structured Outputs**: Guaranteed schema conformance using Pydantic models
- **Vision**: Reads images, PDFs, scans; token cost = (width x height) / 750
- **Prompt Caching**: Cache system prompts, pay 0.1x on subsequent calls
- **Batch API**: 50% discount, async, up to 100K requests per batch
- **Extended Thinking**: Scratchpad for complex reasoning (fraud analysis, matching)
- **Files API**: Upload once, reference by file_id in multiple requests

### Production Architecture
- API Gateway for rate limiting and auth
- Invoice Queue (Redis/SQS) to decouple ingestion from processing
- Worker Pool consuming from queue, calling Claude API
- Model Router for Haiku/Sonnet/Opus based on complexity
- PostgreSQL for extracted data, S3 for documents
- Retry logic with exponential backoff on 429/529 errors

## 6. System Prompts Explanation

User asked: "When would I give those system prompts to you?"

### Two Different "Claudes"
- **Claude Code (me)**: Development partner who writes the code
- **Claude API**: The AI engine inside the product, called programmatically

### How System Prompts Work
The prompts are hardcoded into the application code. They're sent to the Claude API every time the app processes an invoice:

```
1. Customer uploads invoice to the app
2. Backend code runs
3. Code calls Claude API with:
   - System prompt (baked into code)
   - Invoice image (from upload)
   - Output schema (Pydantic model)
4. Claude API returns structured JSON
5. App stores results and shows to customer
```

### Where Prompts Live in the Codebase
```
src/
  prompts/
    invoice_extraction.py    <- Prompt 1
    gl_coding.py             <- Prompt 2
    fraud_detection.py       <- Prompt 3
    aia_billing.py           <- Prompt 4
    lien_waiver.py           <- Prompt 5
    po_matching.py           <- Prompt 6
  services/
    invoice_processor.py     <- Calls Claude API with those prompts
```
